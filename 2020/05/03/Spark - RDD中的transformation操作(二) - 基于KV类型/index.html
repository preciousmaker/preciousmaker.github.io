基于KV类型的transformation操作

RDD编程场景可能不多，但是面试会问
关于资源问题，大公司充足，能用硬件解决的就用硬件解决，调优的力度小
中小型公司不够，只能去调优，先看partition，再看task

mapValues

Pass each value in the key-value pair RDD through a map function without changing the keys;this also retains the original RDD’s partitioning.
在不更改key的情况下，通过映射函数传递键值对RDD中的每个value；这也保留了原始RDD的分区。

意思是只对value进行map操作

flatMapValues

Pass each value in the key-value pair RDD through a flatMap function without changing the keys; this also retains the original RDD’s partitioning.
通过flatMap函数传递键值对RDD中的每个值，而无需更改键； 这也保留了原始的RDD分区。

意思是只对value进行flatMap操作

keys

Return an RDD with the keys of each tuple.
返回RDD中每个元组的key。

values

Return an RDD with the values of each tuple.
返回RDD中每个元组的values。

keyBy

Creates tuples of the elements in this RDD by applying f.
通过函数f在该RDD中创建元素的元组。

单value的RDD转成KV类型

reduceByKey

因为有shuffle，所以拆分成了两个stage，对应MR就是两个MR作业

WC中reduceByKey后是 RDD[(String, Int)]

This will also perform the merging locally on each mapper before sending results to a reducer, similarly to a “combiner” in MapReduce.
这也将在将结果发送到reducer之前在每个mapper上本地执行合并，类似于MR中的combiner，聚合过后的数据走shuffle

groupByKey

WC中groupByKey后是 RDD[(String, Iterable[Int])]，value的结果默认是 CompactBuffer ，这种类型是 extend Seq 的
WC中的 .reduceByKey(+) 也可以被替换为 .groupByKey().mapValues(_.size)

Group the values for each key in the RDD into a single sequence.
将 RDD 中每个键的值分组为一个序列。
@note This operation may be very expensive.
注意 这个操作可能是非常昂贵的
源码中，groupByKey也调用的是combineByKeyWithClassTag

这里的mapSideCombine = false，所以不会在mapper预聚合，数据量可能非常大，也就是所有数据都走shuffle

groupBy

由于没有了ByKey，所以可以按照自己想要的情况进行分组

按照自己本身分组

按照字符长度分组

按照奇偶数分组

其实底层还是调用groupByKey，不过这个key可以自己设定，也就是map把单value的转成KV的形式(map(t => (cleanF(t), t)))

WC中的 .map((,1)).groupByKey().mapValues(.size) 也可以被替换为 .groupBy(x => x).mapValues(_.size)

sortBy

字面意思，排序，默认升序

倒序

distinct(去重)

distinct 有shuffle，也可以强行修改分区数

强行修改分区数

map :
output:(key, nullWritable)

reduce :
input:(key, Iterable(nullWritable,nullWritable,nullWritable,…))
output:去value，仅取key就可以

交并差

union

并集，不去重，这点和sql中的union不一样，类似union all
分区数为两个分区数相加，也就是没有shuffle

intersection

交集，去重，有shuffle
分区数为两个分区数中最大的那个

subtract

差集，不去重，但是有shuffle
a.subtract(b) a有b没有
分区数为a的分区数

cartesian 笛卡尔积(慎用)

无shuffle，分区数为两者相乘

join

默认是inner join

分区数为两个分区数中最大的那个

leftOuterJoin

rightOuterJoin

fullOuterJoin

所有的join，底层都调用的是 cogroup
