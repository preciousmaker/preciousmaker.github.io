也许很多Linux命令较为抽象不易记忆，但是好记性不如烂笔头，记下来加深记忆，随时补充。

Linux常用命令

pwd

全称：Print Working Directory
查看当前光标所在目录(路径 path)

结果为绝对路径

ls

显示的文件夹和文件的名称

可以看到权限 用户用户组 时间 大小等额外信息

可以显示隐藏文件夹和文件(隐藏文件夹和文件一般为.开头)

仅仅查看文件大小，文件夹不行

按时间升序显示
-r 反转排序顺序
-t 按时间排序(默认降序)

mkdir

创建文件夹

并联创建文件夹

并联创建文件夹，此处-p为递归创建目录

cd

全称：Change Directory
切换目录(路径 path)

切换目标路径

切换到家目录

切换到上一级目录

回退到上一次命令的文件夹(常用)

路径

绝对路径：/根目录开头，路径是齐全的
相对路径：不是以/开头，路径是短的

help

在使用命令的时候有些不熟练，可以通过 –help 去查看如何使用该命令

Usage: mkdir [OPTION]… DIRECTORY…
其中[OPTION] 代表可选项
… 代表可以有多个参数
DIRECTORY 类似单独的大写字母代表必填项

man

相比于help，是更为详细的命令帮助

clear

清理屏幕

mv cp

mv 移动，始终是一份，快，类似于windows的剪切粘贴
cp 拷贝，最终有两份，慢，类似于windows的复制粘贴

创建文件

touch

创建一个空文件【推荐】

vi

可以创建一个空文件，默认进入命令行模式
INSERT键：进入编辑模式
ESC键：从编辑模式变为命令行模式
:键 ：从命令行模式变为尾行模式
数据wq保存(w保存，q退出)

:wq 保存退出
:wq! 强制保存退出
:q 强制退出

如果怕错，可以先备份一下，例如 cp conf conf20201108

如果进入提示是swap file，只需要ll -a查看，然后进行删除即可
其中粘贴的坑，必须进入编辑模式，否则第一行会丢失数据【生产巨坑】

在命令行模式，常用的快捷方式：
dd 删除当前行
dG 删除当前行及以下所有行
ndd 删除当前行及以下n-1行
gg 跳转到第一行的第一个字母
G 跳转到最后一行的第一个字母
$ 跳转到行位

行号
尾行模式 set nu
去除行号 set nonu

比如要cd到一个很深的目录，可以使用alias p=’cd /…/…/…/…’

1.passwd文件中的
hadoop:x:1000:1000::/home/hadoop:/bin/bash
被换成
hadoop:x:1000:1000::/home/hadoop:/sbin/nologin
显示：This account is currently not avaliable

2.被换成
hadoop:x:1000:1000::/home/hadoop:/bin/false
则不准登录，或者切换用户没起作用

例如CDH部署，hdfs，hbase等用户，su - hdfs切不过去，原因就在 /bin/false
jps命令查看都是不可用的
su - hdfs之前，将/bin/false改为/bin/bash即可

权限

1.d rwx r-x r-x
2.- rw- r– r–
第一个字符 d 代表文件夹 - 代表文件
接下来三个字母分别代表 读r4 写w2 执x1 -没有任何权限
第一组：rw- 文件或文件夹的所属用户，只有读写权限，没有执行权限
第二组：r– 文件或文件夹的所属用户组的成员，只有读权限
第三组：r– 其他用户组的成员对这个文件或文件夹的权限，只有读写权限

权限错误的根本原因，当前用户对文件或文件夹的操作，不符合其拥有的权限

权限命令

chmod -R 777 文件或文件夹
chown -R 用户或用户组 文件或文件夹
这是两个为数不多的经常会使用的-R参数的命令

a.计算程序 hive sql、spark 、flink 密集计算 是不是要调优

端口号

过滤条件为pid或者名称均可
例如 netstat -nlp | grep 4211 或者 netstat -nlp | grep java

总结：
有进程PID 不一定就有端口号
服务的通信交流，其实就是需要 ip+端口号

面试题：打开某机器xxx服务的web界面
ifconfig 得到ip
ps -ef|grep xxx 找到pid
netstat -nlp| grep pid 找到对应端口号
