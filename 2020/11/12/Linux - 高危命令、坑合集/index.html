坑

1. vi命令，粘贴的坑

2.echo

如果想要清空文件，不能使用以下指令

此命令可以清空文件内容，但是并不是真正的清空，该文件还存在一个字节，慎用

推荐使用

将1.log文件置为空文件，清空文件内容 推荐

3. 个人环境变量

场景：ssh远程执行B机器，启动服务命令，报错：java command not found；但是直接登陆B机器，命令可以找到
https://blog.csdn.net/whitehack/article/details/51705889
结论：尽量使用.bashrc而不是.bash_profile

4. rm

场景：如果变量LOG_PATH不存在，那么
rm -rf ${LOG_PATH}/* 等价于 rm -rf /*，非常致命
所以需要在每次删除之前，需要判断${LOG_PATH}是否存在
或者在shell脚本上，第二行加上set -u，脚本在头部加上它，遇到不存在的变量就会报错，并停止执行。 【推荐】

高危命令

1. echo

如果覆盖的是环境变量或重要的配置文件，将会出大问题

2. rm

不提示，直接删除 **【高危命令】**
