Hive简介

Hadoop广义是大数据生态圈，其中Hive是使用SQL完成大数据统计分析的工具
Hadoop狭义是Hadoop软件，包括HDFS，MapReduce，YARN

Hive是Facebook公司开源的工具，用来解决海量的结构化日志的统计问题
Hive是构建在Hadoop之上的数据仓库
HDFS：Hive的数据存放在 distributed storage ，元数据存放在 metastore 所对应的底层数据库中的(一般是MySQL)
MapReduce：Hive的作业(SQL)是通过Hive的框架翻译成MapReduce作业
这里的引擎也可以是Tez，Spark
不管底层是使用什么引擎，对于用户来说是不感知的
同样的SQL，只需要通过一个参数切换，就可以运行不同的执行引擎之上
YARN：Hive的作业是提交到Yarn上面去运行的

Hadoop开发可以使用单机，但是生产环境一定是分布式
Hive其实就是一个客户端而已，没有集群的概念，提交作业到集群的Yarn上去运行(没有感情的提交机器)
SQL ==> Hive ==> MapReduce => YARN
生产环境上，哪台机器需要提交Hive，就在哪台机器上配置Hive，不同机器上的Hive相互独立

Hive职责：将SQL翻译成底层对应的执行引擎作业

metastore

统一的元数据(metadata)管理
元数据：描述数据的数据，也就是schema
源数据：来源系统的数据，HDFS上的数据，各个数据库的数据等
例如：
源数据：HDFS上有一个文件user.txt
1,andy,20
2,bob,30
3,candy,25
元数据：
table：user_infomation
id：int
name：string
age：int
sep(分隔符)：,
列的顺序：例如id是第一个，name是第二个，age是第三个

Spark SQL/Impala/Presto等，统一使用metastore，也就是说Hive建的表，Spark SQL也可以使用

官方描述

The Apache Hive ™ data warehouse software facilitates reading, writing, and managing large datasets residing in distributed storage using SQL.
Structure can be projected onto data already in storage.
A command line tool and JDBC driver are provided to connect users to Hive.

distributed storage(分布式存储)：
HDFS，AWS S3，OOS(阿里云)，COS(腾讯云)
这些系统，Hive均可以对接，都有对应的jar包，本地文件系统(file开头)也可以对接

Hive vs RDBMS(关系型数据库)

分布式：均支持
节点数：均支持
成本： 廉价 vs 昂贵
数据量：TB,PB vs GB
DML: 0.14版本后支持(但是不建议用) vs 支持
事务： 均支持
延时性：高 vs 低

Hive适用场景

批处理/离线处理
延时性很大的
尽量少涉及到update或者delete这种操作，虽然是支持的

Hive优缺点

优：易上手、比MapReduce使用起来简单的多
缺：高延时

初学者容易问的问题

Q:Hive的执行速度对于比MySQL的数据谁快谁慢？
A:詹姆斯和C罗谁的成就高？不是一个领域的，没有对比性

Q:Hive SQL和MySQL的SQL有啥关系？
A:没有任何关系

修改配置

可以在hive-site.xml中修改配置，也可以通过其他方法去修改配置

hive –hiveconf

hive启动的时候可以跟上修改配置的参数

set

hive.cli.print.header=true
