用户创建的数据库，比如我创建的是 precious_test，对应的HDFS地址为：/user/hive/warehouse/precious_test.db
如果建库的时候指定了LOACTION，那么对应的HDFS地址就是LOCATION地址，例如：

删除库

使用ALTER和DROP，来修改和删除库

这时候再去查HDFS，仅剩/dw_dir，没有test目录

默认情况下,Hive不允许删除包含表的数据库，有两个解决办法：
1.手动删除库下所有表，然后删库
2.使用cascade关键字
drop database if exists t1 cascade;

创建表

官方建表语句
https://cwiki.apache.org/confluence/display/Hive/LanguageManual+DDL#LanguageManualDDL-CreateTable

可以看到非常长，但是不用怕，常用的多写几遍就记得住，不常用的可以去官网查，毕竟建表是非常谨慎的操作

CREATE TABLE：创建一个指定名字的表，如果相同名字的表已经存在，则抛出异常；用户可以用IF NOT EXISTS 选项来忽略这个异常
EXTERNAL：可以让用户创建一个外部表，在建表的同时指定一个执行实际数据的路径(LOCATION)
col_name data_type：列名以及对应的数据类型，多个列之间使用逗号分隔
COMMENT：为表与字段增加描述
PARTITIONED BY：指定分区
CLUSTERED BY：对于每一个表或者分区，Hive可以进一步组织成桶，Hive采用对列值哈希，然后除以桶的个数求余的方式决定该条记录存放在哪个桶当中。
ROW FORMAT：一般是 ROW FORMAT DELIMITED FIELDS TERMINATED BY ‘,’，这里指定表存储中列的分隔符，默认是 \001，这里指定的是逗号分隔符，还可以指定其他列的分隔符。
STORED AS：STORED AS SEQUENCEFILE|TEXTFILE|RCFILE，如果文件数据是纯文本，可以使用 STORED AS TEXTFILE，如果数据需要压缩，使用 STORED AS SEQUENCEFILE。
LOCATION：定义Hive表的数据在HDFS上的存储路径，如果定义的是外部表，则需要直接指定一个路径；内部表不用指定，默认路径/user/hive/warehouse
AS select_statement: 通过select的sql语句的结果来创建表

内部表 vs 外部表

有诸多不同，但是最主要的是
内部表(MANAGED)：删除表之后，HDFS和MySQL的数据都被删除
外部表(EXTERNAL)；删除表之后，MySQL的数据被删除，但是HDFS的数据还存在

外部表的不删除HDFS上的数据这一特点，方便数据的共享，比如各类原始数据，日志等
内部表可以作为中间表和结果表，存储自己的结果

时间相关

functions description example
current_date 年月日 select current_date;
current_timestamp 年月日时分秒毫秒 select current_timestamp;
unix_timestamp 到1970-01-01 00:00:00.000的秒数 select unix_timestamp(current_timestamp);
from_unixtime 格式化时间 select from_unixtime(unix_timestamp(current_timestamp),’YYYY-MM-dd’);
to_date 返回字符串的日期部分 select to_date(‘2020-12-12 09:00:00.234’);
year/month/day/hour/minute/second 返回年/月/日/时/分/秒 select year(current_timestamp),month(current_timestamp),day(current_timestamp);
weekofyear 返回这是年的第几周 select weekofyear(current_timestamp);
dayofmonth 返回这是月的第几天 select dayofmonth(current_timestamp);
months_between 两个日期相差的月份 select months_between(current_timestamp,’2030-09-09’);
add_months 增加月 select add_months(current_timestamp,3);
datediff 两个日期相差的天数 select datediff(current_timestamp,’2030-09-09’);
date_add 增加天 select date_add(current_timestamp,3);
date_sub 减少天 select date_sub(current_timestamp,3);
last_day 月最后一天 select last_day(current_timestamp);
trunc 返回按格式截取的日期 select trunc(current_date,’YYYY’);

functions description example
