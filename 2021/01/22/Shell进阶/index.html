脚本的规范

脚本后缀默认一般是 .sh
第一行是解释器 #!/bin/bash

编写一个简单的输出”hello！”的脚本

其中 sh hello.sh不赋予x权限就可以执行
并且第一行不写 #!/bin/bash 也可以直接执行
并且不关心是绝对路径还是相对路径
【推荐使用】

./hello.sh 没有执行权限不能执行，需要赋权，一般是相对路径

使用绝对路径也是可以的/root/shell/hello.sh ，不过同样需要赋权

debug

编写一个简单的脚本

sh -x hello.sh可进行debug，查看每段语法的结果，一般用于开发测试阶段

也可以在脚本中设置#!/bin/bash -x

不过这样的话，sh语法就不会debug，感觉上是跳过了第一行的解释器

IFS 是一种 set 变量，当 shell 处理”命令替换”和”参数替换”时，shell 根据 IFS 的值，默认是 space, tab, newline 来拆解读入的变量，然后对特殊字符进行处理，最后重新组合赋值给该变量。
