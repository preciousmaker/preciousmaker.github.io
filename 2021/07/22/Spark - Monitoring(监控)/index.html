Spark - Monitoring(监控)

发表于2021-07-22|更新于2021-07-22|Spark
|阅读量:

官网

WebUI 默认4040，如果同时启动多个，那么查到4040被占用就会+1，使用4041端口，以此类推

这些信息只有在 application 运行期间可以访问到，sc.stop 之后网页就就访问不到
其实这些信息是以 json 形式存在 eventlog 中的，WebUI只是解析展示

如果想要运行结束后去看这些信息，可以在启动 application 之前将 spark.eventlog.enabled 设置为true

作用

1.A list of scheduler stages and tasks
stage和task列表

2.A summary of RDD sizes and memory usage
RDD 大小和内存使用的总结(cache)

3.Environmental information
环境信息

4.Information about the running executors
有关正在运行的执行程序的信息

./sbin/start-history-server.sh
启动一个进程，创建一个WebUI 默认18080，用来展示未完成和已完成的 application

相关参数

常规参数

spark的一些参数可以配置(默认)在 conf/spark-defaults.conf 中，例如
spark.eventLog.enabled true 打开
spark.eventLog.dir hdfs://… 运行中的日志往哪个文件夹中记录，格式为json格式，UI上的信息都是从这个json文件中来的

spark.history.*

如果是spark.history.*的配置，需要配置在 SPARK_HISTORY_OPTS 环境变量中，具体在spark-env.sh中
export SPARK_HISTORY_OPTS = “-Dspark.history.fs.logDirectory=hdfs://… -Dspark.history.ui.port=7777”
这里使用的都是-Dx=y的形式，例如
spark.history.provider 不用配置 默认值即可
spark.history.fs.logDirectory hdfs://… 从哪个文件中去读取日志，一般和 spark.eventLog.dir 设置一致，才可以读到
spark.history.ui.port 默认是上面指定的18080，可以自己修改，例如改成7777

spark.history.fs.update.interval 10s 默认10s刷新一次
spark.history.retainedApplications 50 默认为50，意思是缓存50个application的信息到内存，动态刷新，如果要看比较老的application，要从磁盘去读
spark.history.ui.maxApplications Int.MaxValue 默认记录的历史application的个数
spark.history.fs.cleaner.enabled false 需要周期清理的话，改为true，不清理的话东西太多，HDFS扛不住
spark.history.fs.cleaner.interval 1d 默认一天清理一次
spark.history.fs.cleaner.maxAge 7d 默认一次清理7天的数据

One way to signal the completion of a Spark job is to stop the Spark Context explicitly (sc.stop())
有一种让 spark 作业显式停止的方法是 sc.stop()
只有在 application 运行期间可以访问到

REST API

Spark提供了非常多接口，可以直接在网页获取json格式的数据，有的公司改一改就成了自己的平台
percious:7777/api/v1/applications 获取application的信息

其他详细的接口可以访问文章开头的官网查看
