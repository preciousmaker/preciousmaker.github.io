MapReduce编程 - 序列化及其应用

发表于2021-03-01|更新于2021-03-09|MapReduce
|阅读量:

序列化：内存中对象 ==> 字节数组 实现网络传输、数据持久化
反序列化：字节数组 ==> 内存中

在hadoop编程中会出现 NotSerializableException 不能序列化异常
类似这种错误，需要在被序列化的类中 implements Serializable

Person 类

类中定义了四个字段
实现了其带参 / 不带参的构造函数
实现了get / set方法
重写了toString()方法

ObjectOutputStream / ObjectInputStream

这是JDK自带的序列化 / 反序列化框架
ObjectOutputStream 最终实现的是 ObjectOutput 的 writeObject 方法
ObjectInputStream 最终实现的是 ObjectInput 的 readObject 方法
序列化后的文件：d:/tmp/serde/person.obj 132 字节

DataOutputStream / DataInputStream

还有这种序列化框架，底层实现的是 DataOutput / DataInput
序列化后的文件：d:/tmp/serde/person2.obj 22 字节
远远小于 ObjectOutputStream 序列化后的文件大小

反序列化的时候注意顺序不能错

Writable

hadoop 中常用的 Writable 实现的也是 DataOutput / DataInput
Writable 是 hadoop 序列化框架的顶层接口，定义了
void write(DataOutput out)
void readFields(DataInput in)

接下来需要实现一个需求：
在access.log中，计算每个手机号的上行流量之和、下行流量之和、总流量
注：第二列为手机号
倒数第三列为上行流量
倒数第二列为下行流量

access.log

Access.java

首先自定义MR中的序列化对象

  1. 实现Writable接口
  2. 重写write和readFields方法，注意顺序和类型
  3. 建议重写toString方法
  4. 建议加上有参和无参构造器

这里implements Writable
使用 write() 和 readFields() 来实现序列化 / 反序列化

AccessDriver.java

虽然名字是 Driver ，但为了方便也把 Mapper 和 Reducer 也写在里面了
