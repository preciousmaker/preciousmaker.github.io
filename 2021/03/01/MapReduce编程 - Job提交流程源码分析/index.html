org.apache.hadoop.mapreduce.InputFormat

说白了就是读数据的

InputFormat 源码

InputFormat describes the input-specification for a Map-Reduce job.
InputFormat 描述了 Map-Reduce 作业的输入规范。

InputSplit？Mapper？进程？block？

Split-up the input file(s) into logical InputSplit, each of which is then assigned to an individual Mapper.
将输入文件按逻辑拆分为 InputSplit ，然后将每个 InputSplit 分配给一个单独的 Mapper 。

一个 InputSplit 对应一个 Mapper ，也就对应一个进程，也就对应一个 JVM ，默认情况下等于一个block
但问题是 windows 的 blockSize 是32M(服务器上是128M)，所以如果读的是一个大于32M的文件，就会分成多个 InputSplit ，对应多个 Mapper ，也就有多个进程
或者输入的有多个文件，也会生成多个 InputSplit
也就是 InputSplit 决定了 MapTask 的并行度，决定了有几个进程同时在运行
一般来说，资源够的情况下，并行度越高越好

32M： org.apache.hadoop.fs.FileSystem

所以其实整个流程是：
Input ==> InputSplit(s) ==> Mapper ==> Shuffle ==> Reducer ==> Output

org.apache.hadoop.mapreduce.lib.input.FileInputFormat

AccessDriver.java中，读数据是用 FileInputFormat ，其继承了 InputFormat ，是一种基于文件的 InputFormat

还有其他继承了 FileInputFormat ， 例如
TextInputFormat
CombineFileInputFormat
KeyValueTextInputFormat
NLineInputFormat
SequenceFileInputFormat

org.apache.hadoop.mapreduce.lib.input.TextInputFormat

TextInputFormat 纯文本文件的InputFormat。文件按行进行split。换行或回车符都用来表示行结束。key是文件中的位置，value是文本行。
比较常用

org.apache.hadoop.mapreduce.lib.input.NLineInputFormat

NLineInputFormat which splits N lines of input as one split.
NLineInputFormat将N行输入拆分为一个拆分。

默认值为1，也就是1行拆分为一个split

在使用的时候，在Driver中设置
job.setInputFormatClass(NLineInputFormat.class);
就会按照行进行split

如果想按照N行进行split
NLineInputFormat.setNumLinesPerSplit(job, N);
