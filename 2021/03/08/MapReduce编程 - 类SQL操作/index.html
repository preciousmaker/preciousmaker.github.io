Reduce Join / Common Join / Shffule Join

这是最常见的join操作

Mapper端 : join的条件作为key，来自于不同的表的数据(Info)需要打上flag
Reducer端 : 实现 join ，一个Reducer中只会有相同的key，按flag将不同的Info进行赋值，再输出

这种方法，一定有Shuffle
开发过程中，大部分情况下，能不使用Shuflle就不使用Shuffle

Map Join / Broadcast Join

有一方是小表的情况下可以使用Map Join
真正的join操作是在Map端完成，不需要Reducer，那就说明没有Shuffle

工作原理

小表加载到缓存中
在读大表的每行数据时，直接从缓存中进行join

但是，小表不能太大，因为要加载到缓存中，过大会OOM
