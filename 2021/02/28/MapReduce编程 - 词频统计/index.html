MapReduce 编程简介

官网文档地址：
https://hadoop.apache.org/docs/stable/hadoop-mapreduce-client/hadoop-mapreduce-client-core/MapReduceTutorial.html

MapReduce - “easily” writing applications

“easily” writing applications(“轻松”编写应用程序)
对于如何容错，RPC通信，开发人员不用去关注，关注我们自己的业务逻辑就可以，这方面来说，easily
业务逻辑 + MR框架自带的内置组件 ==> 分布式应用程序开发

但是用 MapReduce 来做应用程序的开发，是非常麻烦的
Map : 映射 把一个任务拆解成多个
Reduce : 聚合 把拆解开的任务做最后的聚合操作

MapReduce - 适用/不适用场景

适用场景 : 离线、批处理计算
不适用场景 : 实时计算、迭代计算(很多作业串起来的计算)

MapReduce - 运行中的进程

MapReduce 运行过程中会产生以下进程：
MRAppMaster
YarnChild : Task
不管是MapTask还是ReduceTask，都是以进程的方式运行的，要申请资源，释放资源，这是MR慢的一个原因

序列化

Hadoop内置的数据序列化类型，因为Java自带的序列化比较慢
boolean ==> BooleanWritable
string ==> Text
int ==> IntWritable
double ==> DoubleWritable

MR编程核心要素

  1. Mapper ==> org.apache.hadoop.mapreduce.Mapper
  2. Reducer ==> org.apache.hadoop.mapreduce.Reducer
  3. Driver ==> 连接 Mapper 和 Reducer ， 提交成job

打包运行在服务器

调整代码

需要略微修改代码，调整 input 和 output

打包，复制到服务器

打包放到服务器上运行，点击右侧Maven，右键package ==> Run Maven Build
maven打包运行
瘦包：仅仅只包含源码 比较小的
胖包：打入相关的依赖 挺大的
所以pom.xml中的hadoop-client不需要打包，因为服务器上都有

生成的jar包在工作空间的target目录下，这里是瘦包，复制到服务器的/home/hadoop/lib

hadoop jar命令运行

使用 hadoop jar gulu-1.0.jar com.precious.hadoop.mapreduce.wordcount.WordCountDriver_Server /hdfsapi/wc.txt /hdfsapi/wc_mr_result
将hdfs上的wc.txt文件作为input，wc_mr_result作为output

查看结果

运行结束后/hdfsapi/wc_mr_result文件夹中会有两个文件
_SUCCESS 标志成功
part-r-00000 真正存放结果的文件
使用 hdfs dfs -cat /hdfsapi/wc_mr_result/part-r-00000 可以查看结果

使用脚本运行

或者写一个shell脚本 /home/hadoop/shell/wc.sh
写完记得 chmod +x wc.sh
